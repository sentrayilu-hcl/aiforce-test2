# Manual Test Cases - US-001 Secure Login and Session Management

Traceability: `BN-01 -> BR-001 -> EPIC-001 -> US-001`

| Test Case ID | Title | Objective | Preconditions | Test Steps | Expected Result | Priority | Traceability |
|---|---|---|---|---|---|---|---|
| TC-US001-001 | Successful login creates authenticated session | Verify valid credentials generate secure session and token | User account exists and is active | 1. Send login request with valid username/password 2. Capture response 3. Verify access token/session ID present 4. Call protected endpoint with token | Login succeeds, token and session ID returned, protected endpoint accessible | High | BN-01 -> BR-001 -> EPIC-001 -> US-001 |
| TC-US001-002 | Invalid credentials are rejected | Verify invalid login is denied with proper error | User account exists | 1. Send login request with invalid password 2. Observe response | 401 Unauthorized with appropriate error message | High | BN-01 -> BR-001 -> EPIC-001 -> US-001 |
| TC-US001-003 | Inactive session expires after timeout | Verify session becomes invalid after inactivity window | Active session exists | 1. Login successfully 2. Wait until configured timeout elapses 3. Call protected endpoint using same token/session | Session is expired and protected access is denied, re-authentication required | High | BN-01 -> BR-001 -> EPIC-001 -> US-001 |
| TC-US001-004 | Logout invalidates active session | Verify logout revokes session immediately | Active session exists | 1. Login successfully 2. Call logout with session ID 3. Call protected endpoint with same token/session | Logout succeeds, subsequent protected access denied | High | BN-01 -> BR-001 -> EPIC-001 -> US-001 |
| TC-US001-005 | Expired session requires re-authentication | Verify expired session cannot access protected functionality | Expired session exists | 1. Use expired token/session 2. Call protected endpoint | 401 Unauthorized, user prompted to log in again | High | BN-01 -> BR-001 -> EPIC-001 -> US-001 |
| TC-US001-006 | Invalid session ID is rejected | Verify missing/invalid session ID cannot validate or logout | No valid session or tampered session ID | 1. | TC-US001-007 | Protected endpoint rejects missing Authorization header | Verify security enforcement on protected APIs | User not authenticated | 1. Call protected endpoint without Authorization header | 401 Unauthorized | High | BN-01 -> BR-001 -> EPIC-001 -> US-001 |
| TC-US001-008 | Protected endpoint rejects malformed Bearer token | Verify malformed token is blocked | User not authenticated | 1. Call protected endpoint with malformed Authorization header | 401 Unauthorized | Medium | BN-01 -> BR-001 -> EPIC-001 -> US-001 |
| TC-US001-009 | Session validation endpoint confirms active session | Verify validate API works for active session | Active session exists | 1. Call session validate with active session ID | Response indicates valid=true and correct username/session ID | Medium | BN-01 -> BR-001 -> EPIC-001 -> US-001 |
| TC-US001-010 | Re-login after logout creates new session | Verify re-authentication after logout works | User logged out from active session | 1. Login 2. Logout 3. Login again | New session/token created and access restored | Medium | BN-01 -> BR-001 -> EPIC-001 -> US-001 |
